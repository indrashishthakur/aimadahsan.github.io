Generating clean URLs on Pelican

Fri 22 January 2016

Filed under Tutorials

Tags pelican tutorials

Around last week I started working on this blog and while most of the things were easy to setup, there is one thing that got me stumped for a while and I figured I should share how I worked my way around it. So, in this small tutorial, I'm going to share how to generate clean URLs on your Pelican blog.

By default, the path to the URLs generated by Pelican for your posts are of the form: 

./{slug}.html

This means that your URL to some post might look something of sort:

http://domain.com/my-post-slug-here.html

Now, on the address bar it looks quite ugly. For the sake of asthetic apperance of your URLs, you can change this with just few lines. Let me show you how.

Go to pelicanconf.py file and add the following two lines to it:

ARTICLE_URL = "post/{date:%Y}/{date:%m}/{slug}/"
ARTICLE_SAVE_AS = "post/{date:%Y}/{date:%m}/{slug}/index.html"

If you generate your blog now, the parth will be generated as follows:

./post/*year-here*/*month-here*/*slug-here*

Note that this is just how I have my URLs customised. You can always change it to suit your needs. For customizing the date and month or to add other fields, you can refer to the table here.

This customization is only for the article. We can also clean up URLs for pages, authors, tags and categories by customizing the following variables: 

CATEGORY_URL = ""
CATEGORY_SAVE_AS = ""

TAG_URL = ""
TAG_SAVE_AS = ""

AUTHOR_URL = ""
AUTHOR_SAVE_AS = ""

PAGE_URL = ""
PAGE_SAVE_AS = ""

You can also customize URLs to your archives too by editing the following variables:

YEAR_ARCHIVE_SAVE_AS = ''
MONTH_ARCHIVE_SAVE_AS = ''
DAY_ARCHIVE_SAVE_AS = ''

I don't feel that there is really need to explain much because the variable names speaks for themselves. You can also browse the full reference here.

If you're lazy and like how URLs are generated on this blog, you are free to copy my configuration for the URL cleanup. Just copy the following code and append it to your pelicanconf.py:

# Formatting to obtain clean URLs
ARTICLE_URL = "post/{date:%Y}/{date:%m}/{slug}/"
ARTICLE_SAVE_AS = "post/{date:%Y}/{date:%m}/{slug}/index.html"

CATEGORY_URL = "category/{slug}/"
CATEGORY_SAVE_AS = "category/{slug}/index.html"

TAG_URL = "tag/{slug}/"
TAG_SAVE_AS = "tag/{slug}/index.html"

AUTHOR_URL = 'author/{slug}/'
AUTHOR_SAVE_AS = 'author/{slug}/index.html'

PAGE_URL = 'page/{slug}/'
PAGE_SAVE_AS = 'page/{slug}/index.html'

# Formatting for yearly archives
YEAR_ARCHIVE_SAVE_AS = 'archive/{date:%Y}/index.html'

I hope you found my post helpful in some way. If you have some questions and suggestions, comment below and I'll try to answer it as soon as possible. You can also contact me via e-mail or any other social media (check out About page for links).
